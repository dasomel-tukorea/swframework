XSS 방어 체험 — 4가지 방법 비교

방어 방법 비교: ① th:utext / th:text / 수동 이스케이프 / OWASP Sanitizer (현재 페이지)  |  ② XSS Servlet Filter 필터 레벨 방어 →
실습: <b>굵은글씨</b><script>alert('XSS')</script> 를 입력하면 4가지 방법의 차이를 한눈에 볼 수 있습니다.
목록으로

출력 결과 비교

① th:utext — 위험 (이스케이프 없음)
th:utext="${userInput}"
출력값
스크립트가 실행됩니다! 사용자 입력에 절대 사용 금지
② th:text — 안전 (Thymeleaf 뷰 레이어 이스케이프)
th:text="${userInput}"
출력값
HTML 소스 보기: <script> → &lt;script&gt; 로 변환됨
③ 수동 이스케이프 — 안전 (Controller 레이어에서 처리)
HtmlUtils.htmlEscape(userInput) → th:utext="${manualEscaped}"
출력값
Controller 에서 HtmlUtils.htmlEscape() 로 이스케이프 후 th:utext 로 출력 — 이미 방어된 문자열이므로 스크립트 실행 안 됨
④ OWASP HTML Sanitizer — allowlist 방식 (안전한 HTML 보존 + 위험 태그 제거)
Sanitizers.FORMATTING.and(Sanitizers.LINKS).sanitize(userInput) → th:utext="${owaspSanitized}"
출력값
<b>굵은글씨</b><script>alert('XSS')</script> 입력 시:
→ 굵은글씨 는 그대로 렌더링 (허용 태그 보존)
→ <script> 는 완전히 제거 (allowlist에 없는 태그 차단)

정리

방법 이스케이프 위치 안전 여부 특징
① th:utext 없음 위험 HTML 그대로 렌더링
② th:text View 레이어 (Thymeleaf) 안전 가장 간단, Spring MVC 기본 권장
③ HtmlUtils.htmlEscape() Controller 레이어 (Java) 안전 모든 HTML 차단, View 에 독립적
④ OWASP Sanitizer Controller 레이어 (Java) 안전 allowlist 방식 — 허용 태그 보존, 위험 태그만 제거
XSS Servlet Filter Filter 레이어 (Servlet) 안전 Controller 코드 변경 없이 전역 적용
