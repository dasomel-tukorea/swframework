XSS Servlet Filter — 서블릿 필터 레벨 방어

방어 방법 비교: ← ① th:utext vs th:text vs 수동 이스케이프  |  ② XSS Servlet Filter 레벨 방어 (현재 페이지)
이 페이지의 특징: XssEscapeFilterConfig 의 XssEscapeFilter 가 /students/filter-test URL 에만 등록되어 있습니다.
POST 요청이 오면 Controller 진입 전에 모든 파라미터를 자동 이스케이프합니다.
브라우저 POST → [XssEscapeFilter] → HttpServletRequestWrapper 적용
                 ↓ getParameter() 호출 시 HtmlUtils.htmlEscape() 자동 실행
→ DispatcherServlet → StudentController.filterTest()
                                      ↓ userInput 은 이미 이스케이프된 상태
→ Model → Thymeleaf → HTML 응답
← 이전 비교 페이지

출력값
Controller 의 filterTest() 메서드를 확인하세요 — HtmlUtils.htmlEscape() 호출이 없습니다.
필터가 이미 처리했기 때문에 Controller 코드는 깨끗하게 유지됩니다.
핵심 차이 정리
항목 /students/xss-test (수동) /students/filter-test (필터)
이스케이프 위치 Controller (Java 코드) Servlet Filter (요청 파라미터 레벨)
Controller 코드 HtmlUtils.htmlEscape(userInput) 호출 필요 호출 불필요 (필터가 자동 처리)
적용 범위 해당 메서드만 필터 등록 URL 전체 (자동 전역 적용)
관심사 분리 보안 로직이 비즈니스 코드에 혼재 보안 로직을 필터 계층으로 분리
